1. What did you build? Describe the feature in plain language.
I built a dark mode toggle switch with Claude web.
2. How did micro-iteration feel? Was working in small steps natural or frustrating? Why?
Micro-Iteration felt pretty reasonable, working in small steps felt less overwhelming and more understandable since I'm going
through it step-by-step.
3. What did self-review catch? When you asked the AI to review its own code, what issues did it find? Give at least one specific example.
When I had an issue with the toggle switch, it didn't actually switch between the different modes; it was because it gave
me code that returned null. 
4. Tool impressions. What did you like or dislike about [Copilot Agent / Claude Web]?
Personally I don't enjoy having AI do all my work for me. I appreciated that Claude
Web was able to take me step-by-step through the process of coding and explaining the 
code to me if needed.
5. Self-review patterns. Did the AI consistently catch certain types of issues during self-review
(e.g., edge cases, missing error handling)? Did it ever miss something you caught yourself?
No, it didn't catch many issues; it only caught that one problem when testing the dark mode theme.
6. Browser tool vs. CLI comparison. If you’ve used Claude Code CLI or another terminal tool,
how did the browser-based experience compare? What’s better/worse about each?
The browser-based experience felt smoother since I'm more used to using this version.
The CLI was better with vibe coding.
7. When would you use micro-iteration + self-review? For what kinds of tasks does this workflow make sense?
When would you skip it?
I think this workflow would make sense for strictly planning out as a base before fully coding
a project. I would skip it if I was trying to learn a new feature.
